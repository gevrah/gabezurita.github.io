Ruby Classes vs. JavaScript Constructor Functions

Javascript does not actually have classes, but it has something quite similar: prototypes. When an object is created in Ruby, the behavior of that object is both created and defined by the Ruby class. Unlike Ruby, Javascript has two parts that interact to create Ruby class-like objects, the constructor and the prototype. Constructors only manufacture new objects, but they do not define the behavior of the new object. In Javascript, the prototype is the part the performs the function of defining the behavior of the new object.
